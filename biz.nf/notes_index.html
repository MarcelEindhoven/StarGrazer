Design

Design goals

Portability

Software components should be portable whenever possible. So independent from a particular cloud, databae or other technology. Much waste occurs when companies are forced to switch to a new technology while their software was entangled with existing, obolete software, os, external libraries, hardware.

Frameworks

NodeJS plus JavaScript

The most simple server side implementation is used for user CRUD

HTML plus JavaScript

The most simple client side implementation is used for user CRUD.

Express

Also based on NodeJS plus JavaScript, Express makes implementation of game CRUD simpler by taking care of URL routing and parameter parsing.

Angular

To be implemented for client side game CRUD.

Layering

Front layer is initially in html with javascript. Later frameworks will be added. In this way both local level and high level experience is gained.

REST communication makes the front-end less dependent on which server is used. REST is implemented in the server communication layer.

Logic layer contains gaming logic and cross-player decisions. For simple CRUD operations the logical layer is skipped.

Database layer records all decisions. The database allows players to reenter a game and to replay game-instances for regression testing and trouble shooting.

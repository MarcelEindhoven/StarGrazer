Vocabulary

Player: A player is a participant in a game.
Portal: A portal is an entry page to enter player attributes and to start a game.

Decisions

To save space on the main game page, player attributes are entered in the portal instead of the game page
Player attributes do not change once the game has started.

Each player his own device

Client implementation only knows data visible to player and is responsible for player actions. Server makes all decisions, has all info and synchronizes players. Database allows game to continue after client as well as server restart.

Technology

Pass info between pages

Two technologies are known to pass information directly between pages: form entries and cookies. Indirectly info can be stored on the server.

phone, tablet, PC

Use media queries to distinguish between landscape and portrait and screen size.

Simple animation is parked in css. Complex animation with requestAnimationFrame.
